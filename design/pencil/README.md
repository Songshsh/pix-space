# Pencil 原型协作与设计目录

将每次新增页面的 Evolus Pencil 原型文件（`.pen`）提交到这里，作为页面评审与实现的事实来源。
本项目的页面新增统一采用“先原型后落地”的方式，避免在 Vue 代码里反复试错。

## 1. 约定工作流

1. **需求沟通**：确认信息架构、功能范围、权限边界、关键状态（空/加载/错误/成功）
2. **原型设计**：等你显式确认“可以画图了”后，开始产出 Pencil 文件（`.pen`）
3. **原型评审**：以 Pencil 文件为准做评审与迭代，直到你确认“原型没问题”
4. **代码还原**：等你显式确认“可以将 pencil 文件还原为 vue 文件了”后，再开始将原型还原为 Vue 页面与组件

## 2. 文件位置与命名

- 存放目录：`design/pencil/`
- 命名建议：`<route>-<page>-v<版本号>.pen`
  - `<route>`：去掉前导 `/`，多级路由用 `-` 连接（例：`/user/roles` → `user-roles`）
  - 全部使用短横线命名；版本号从 v1 递增；禁止覆盖旧版本文件
  - 示例：`/users` 对应 `users-list-v1.pen`

## 3. 评审交付物

评审阶段的唯一交付物是 `.pen` 文件（确保可以用 Evolus Pencil 打开查看）。
如需要在即时通讯/邮件中快速预览，可额外导出图片，并与 `.pen` 放在同目录下（导出物不作为唯一事实来源）。

## 4. 还原到 Vue 的范围边界

从 Pencil 还原到 Vue 代码时，会按以下优先级落地：

1. 路由与页面骨架（views + router）
2. 页面内拆分的可复用组件（components）
3. 请求与数据流（api + composables + stores）
4. 空/加载/错误等状态与交互细节（按原型对齐）

## 5. 界面设计规范（面向智能助手与设计师）

为了保证 `.pen` 原型文件的高质量，以及能够无缝还原到 Element Plus，所有产出的原型必须严格遵守以下规范。

### 5.0 总则与优先级

- 总则：原型的目标是“可还原、可评审、可落地”，优先表达结构、状态与交互，避免无意义的像素级皮肤重绘
- 优先级（发生冲突时从上到下取舍）
- - 设计变量映射 > 画布与对齐 > 页面组织 > Element Plus 组件映射 > 视觉微调

### 5.1 设计变量映射（必须对齐 tokens.css）

- 间距必须能一一映射到设计变量
  - 4/8/12/16/24/32px 分别对应：`--ds-space-1/2/3/4/5/6`
- 圆角必须映射到设计变量
  - 4/8/12px 分别对应：`--ds-radius-1/2/3`
- 阴影不手工发明新参数
  - 仅允许使用：`--ds-shadow-1/2`
- 颜色只允许使用设计变量语义（禁止随意填写新颜色值）
  - 主色：`--ds-color-primary`
  - 文本：`--ds-color-text-primary/secondary/inverse`
  - 背景：`--ds-color-bg-primary/secondary`
  - 边框：`--ds-color-border`

### 5.2 画布与布局对齐

- 画板尺寸
  - 网页后台类页面默认使用 1440px 宽画板（高度按内容增长）
  - 如需响应式版本，可额外提供 1280px 宽画板作为补充，但必须沿用同一套间距规则
- 布局网格与对齐
  - 所有元素的 x/y 坐标必须为 4px 的倍数，严禁出现小数位
  - 同一列/同一行的元素必须使用同一条对齐线（左对齐/右对齐/上下居中不得混用）
- 页面安全边距（左右一致）
  - 页面内容区（标题、筛选区、表格/列表、卡片）必须使用同一套左右内边距
  - 推荐默认左右内边距为 24px（对应 `--ds-space-5`）
  - 严禁出现“左 24px、右 20px”或不同模块左右边距不一致的情况
- 内容区宽度策略
  - 管理后台：内容区随容器自适应铺满（减去侧边栏与左右内边距），避免居中留白导致信息密度下降
  - 门户/展示：内容区居中，建议设定最大宽度（例如 1200px），并保持左右内边距不变
- 典型对齐基准（强制）
  - 页面标题、面包屑、卡片边缘、表格边缘必须在同一条垂直基准线上
  - 筛选区表单项与操作按钮需要对齐到同一基线，避免出现“按钮上下漂移”

### 5.3 间距与色彩系统

- 间距（禁止魔法数字）
  - 仅允许使用以下间距值：4 / 8 / 12 / 16 / 24 / 32
  - 页面级内边距推荐 24；模块间距推荐 16 或 24；组件内部间距推荐 8 或 12
  - 不允许出现 18、22、30 等不可解释的“微调值”；需要更紧凑时优先改布局，而不是改成非标间距
- 圆角与阴影
  - 圆角仅允许：4 / 8 / 12
  - 阴影不手工发明新参数；需要层级感时优先使用“卡片 + 合理留白”表达
- 色彩（对齐前端设计变量语义）
  - 主色：用于强调性操作（主要按钮、选中态、链接强调），避免在大面积背景使用
  - 文本色：区分主/次信息层级，避免用纯黑铺满页面
  - 背景色：页面底色与卡片底色需保持明确层级（页面灰、卡片白）
  - 边框色：统一使用浅色边框表达分割，不通过“加深边框”制造层级
- 与前端实现一致
  - 若设计稿需新增不在现有设计变量中的颜色/间距，必须先提出新增需求，评审通过后再进入原型与落地阶段
  - 前端设计变量定义见：`src/styles/tokens.css`

### 5.4 Pencil 页面组织（强制）

- 每个路由页面必须对应 Pencil 中一个同名页面（命名同文件名，不带版本号）
- 空/加载/错误/权限四态必须以独立页面呈现：`<page>/empty|loading|error|forbidden`
- 新增/编辑/详情等弹层交互必须分别提供独立页面，并标注触发入口与关闭方式

### 5.5 Element Plus 组件映射

- 原则：优先用现有组件表达交互
  - 能用 Element Plus 组件表达的，禁止在原型里设计“自研控件外观”
  - 原型需要表达的是结构、状态与交互，不是像素级皮肤重绘
- 页面骨架建议（高频）
  - 容器：el-card（承载模块边界与留白）
  - 筛选区：el-form（行内模式）+ 输入/选择/日期等表单项 + 操作按钮区
  - 列表区：el-table / 列表（含分页、排序、空状态）
  - 弹层：el-dialog / el-drawer（新增、编辑、详情）
  - 提示反馈：成功/失败/警告统一用提示的方式表达，不在原型里自定义弹出样式
- 原型必须覆盖的状态（最少集）
  - 空：无数据时的展示与引导（是否提供“去创建/去上传”等操作入口）
  - 加载：列表/详情的加载中态（骨架或加载遮罩即可）
  - 错误：接口失败时的页面策略（提示文案 + 重试入口）
  - 权限：无权限时的入口与提示（跳转/禁用/隐藏策略需要明确）
- 标注规则（降低还原歧义）
  - 对组件做最少但关键的标注：是否可排序、是否多选、校验规则、默认值、禁用条件、分页策略
  - 任何“看起来像按钮”的元素必须明确：主按钮/次按钮/文字按钮，以及是否有二次确认

### 5.6 交付前自检清单

- 文件与版本
  - 文件命名符合：`<route>-<page>-v<版本号>.pen`
  - 如有快速预览图片，已与 `.pen` 放在同目录下
- 画布与对齐
  - 画板宽度为 1440px（如提供 1280px 版本，规则一致）
  - 所有元素 x/y 坐标为 4px 的倍数，无小数位
  - 页面标题、面包屑、卡片边缘、表格边缘使用同一条垂直对齐基准
  - 页面内容区左右内边距一致，默认 24px
- 间距与视觉
  - 全部间距仅使用：4 / 8 / 12 / 16 / 24 / 32
  - 圆角仅使用：4 / 8 / 12
  - 未引入新的“非设计变量”颜色/间距；如需新增已先评审
- 组件与交互
  - 交互均能映射到 Element Plus 现有组件（无自研控件外观）
  - 弹层选择合理（对话框/抽屉），并明确关闭方式与默认焦点
  - “看起来像按钮”的元素已明确按钮类型与是否二次确认
- 状态与边界
  - 已覆盖：空 / 加载 / 错误 / 权限 四种关键状态
  - 已标注：分页策略、排序/筛选能力、多选、表单校验规则、禁用条件、默认值
