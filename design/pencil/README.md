# Pencil 原型协作与设计目录

将每次新增页面的 Evolus Pencil 原型文件（`.epz`）提交到这里，作为页面评审与实现的事实来源。
本项目的页面新增统一采用“先原型后落地”的方式，避免在 Vue 代码里反复试错。

## 1. 约定工作流

1. **需求沟通**：确认信息架构、功能范围、权限边界、关键状态（空/加载/错误/成功）
2. **原型设计**：等你显式确认“可以画图了”后，开始产出 Pencil 文件（`.epz`）
3. **原型评审**：以 Pencil 文件为准做评审与迭代，直到你确认“原型没问题”
4. **代码还原**：等你显式确认“可以将 pencil 文件还原为 vue 文件了”后，再开始将原型还原为 Vue 页面与组件

## 2. 文件位置与命名

- 存放目录：`design/pencil/`
- 命名建议：`<route>-<page>-v<版本号>.epz`
  - 示例：`/users` 对应 `users-list-v1.epz`

## 3. 评审交付物

评审阶段的唯一交付物是 `.epz` 文件（确保可以用 Evolus Pencil 打开查看）。
如需要在 IM/邮件中快速预览，可额外导出 PNG，并与 `.epz` 放在同目录下（导出物不作为唯一事实来源）。

## 4. 还原到 Vue 的范围边界

从 Pencil 还原到 Vue 代码时，会按以下优先级落地：

1. 路由与页面骨架（views + router）
2. 页面内拆分的可复用组件（components）
3. 请求与数据流（api + composables + stores）
4. 空/加载/错误等状态与交互细节（按原型对齐）
