# Design 目录规范

本文件是 `.pen` 原型设计的唯一规则来源。
流程文档见 `design/README.md`。

## 1. 总则

- 原型目标是“可还原、可评审、可落地”
- 原型优先表达结构、状态与交互，不做无意义的像素级皮肤重绘
- 原型必须服务正式工程实现，不默认把页面视为“原型页例外”

## 2. 单一事实来源

- MUST：设计变量的唯一事实来源是 `src/styles/tokens.css`
- MUST：B/C 端主业务界面的交互主题色以 `--el-*` 为唯一来源；原型中如需表达主题态，应按 `--el-*` 的消费语义设计，而不是再发明一套并行品牌色
- MUST：组件映射与工程实现边界以 `docs/FRONTEND_SPEC.md` 为准
- AVOID：在本文件再次复制 spacing/radius/shadow/color 的具体数值表

说明：

- 若需要新增 token，先提出新增方案，再修改 `src/styles/tokens.css`
- 本文件只定义设计协作原则，不维护第二份 token 清单

## 3. 原型组织

- MUST：新增页面或发生信息架构/交互大改的路由页面，必须对应一个可追溯的画板
- MUST：动态路由参数在画板名中替换为可读文字
- MUST：至少覆盖当前任务适用的关键状态；列表/检索类页面默认应补齐空 / 加载 / 错误，受权限影响的页面需补权限态
- MUST：新增 / 编辑 / 详情等弹层需要独立画板，并标注触发入口与关闭方式

## 4. 布局与对齐

- MUST：后台页面默认使用 1440px 宽画板；如补充响应式版本，规则保持一致
- MUST：页面内容区左右内边距一致
- MUST：标题、面包屑、卡片边缘、表格边缘对齐到同一垂直基准线
- SHOULD：元素坐标使用 4px 网格对齐

## 5. 组件映射

- MUST：优先映射到 Element Plus 现有组件，避免设计自研控件外观
- SHOULD：列表优先映射 `el-table` / 列表容器
- SHOULD：弹层优先映射 `el-dialog` / `el-drawer`
- SHOULD：筛选区优先映射 `el-form` 行内布局
- MUST：任何“看起来像按钮”的元素都要标明按钮类型、是否二次确认、是否可用

## 6. 状态与交互

- MUST：显式标注空态、加载态、错误态、权限态
- MUST：删除、批量操作、下拉菜单、更多操作都要给出展开后状态
- SHOULD：标注排序、筛选、多选、分页、默认值、禁用条件
- SHOULD：标注 hover 与选中态，但只写关键交互，不堆砌微观视觉说明

## 7. 评审清单

- [ ] 画板命名符合 `<PageName>/状态` 或等价可读命名
- [ ] 动态路由参数已替换为可读文字
- [ ] 关键状态已覆盖
- [ ] 布局与对齐关系清晰
- [ ] 组件可以映射到 Element Plus 与当前工程结构
- [ ] 使用的间距、圆角、阴影与非主题主链路颜色均来自 `src/styles/tokens.css`
- [ ] 主业务交互主题色遵循 `--el-*` 语义，没有额外定义并行品牌色来源
- [ ] 没有在本文件或原型说明中发明第二套 token 规则
