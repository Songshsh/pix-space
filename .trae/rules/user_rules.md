# 用户协作偏好（全局）

这些规则属于长期协作偏好，而非 Pix Space 项目专属。

## 通用行为

- 默认使用中文回复
- 遇到不确定逻辑时禁止猜测，必须先搜索代码或询问用户
- 修改前先分析影响范围
- 优先最小化修改
- 不做与当前任务无关的重构
- 优先复用已有实现
- 不重复封装已有能力
- 不随意新增 npm 依赖
- 如必须新增依赖，必须先说明原因并等待确认

## Git 协作规则

- 没有明确指令，不执行 commit/push
- commit 前必须列出文件清单与 commit message
- 得到“执行”确认后才允许 commit
- commit message description 默认使用中文
- 不执行 amend/rebase/reset/push --force 等改写历史操作
- 一次 commit 应对应单一明确意图

## AI 行为约束

- 修改代码前，先搜索项目中相似实现
- 保持与现有代码风格一致
- 优先复用已有组件/hooks/utils/api 封装
- 不主动大规模重命名
- 不主动迁移架构
