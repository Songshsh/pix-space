# Code Review 工作流

检查维度（按优先级）：

1. 正确性：边界条件、异常路径、并发/竞态
2. 安全性：敏感信息、鉴权、输入校验
3. 可维护性：分层是否正确、命名、重复逻辑
4. 交互一致性：Element Plus 使用方式、空/加载/错误状态
5. 可测试性：是否容易写测试、是否有回归验证方式

## 输出格式建议

在输出 Review 结果时，必须结构化展示：

1. **🚨 高风险/阻断项 (Blocker)**：代码中存在可能导致崩溃、安全漏洞、严重性能问题的逻辑，必须立即修复。
2. **⚠️ 建议改进项 (Warning)**：代码虽然能运行，但存在代码异味、可维护性差、未遵循项目规范（如滥用全局状态）等问题，建议优化。
3. **💡 优化/最佳实践 (Nitpick)**：关于 TypeScript 类型收窄、Element Plus API 进阶用法、Vue 3 性能优化的善意建议，可选择性采纳。
4. **✅ 闪光点 (Praise)**：对于设计精巧、考虑周全的代码给予正向反馈。

每条反馈必须附带：**所在文件路径及行号范围**、**问题原因**、以及**修改建议（或代码片段）**。
