# Pix Space - 素材图库 C 端前台设计方案

## 1. 项目愿景与定位

Pix Space 将从现有的后台管理风格（DAM/网盘）转型为“双轨制”架构：

- **C端前台（Portal）**：一个现代化的、沉浸式的灵感素材图库社区（类似 Pinterest / 花瓣网）。主打视觉沉浸、灵感发现、画板收集。
- **B端后台（Admin）**：复用现有的后台管理系统布局，供管理员进行图片审核、用户管理、系统配置。

**本次设计方案（MVP阶段）聚焦于 C 端前台的构建。**

## 2. 核心体验原则

- **视觉沉浸 (Visual First)**：采用“现代卡片风”设计，顶部导航栏（无左侧菜单），极浅灰色背景（如 `#f5f6f7`），白色圆角带微阴影的图片卡片悬浮其上。图片是界面的绝对主角。
- **渐进式浏览 (Progressive Disclosure)**：在瀑布流中点击图片，默认弹出“模态框/抽屉”展示大图及快速操作（收藏、下载），保证浏览连续性；提供“进入详情页”的入口供深度浏览（如 EXIF、评论、相似推荐）。
- **UGC 驱动与画板体系 (Boards Collection)**：用户注册后可自由创建多个“画板（Boards）”。用户可以将系统中的公共图片“采集/Pin”到自己的画板，也可以上传自己的私有图片到画板中。用户可选择将私有图片“投稿/发布”至公共图库，经管理员审核后全站可见。

## 3. 架构设计与核心模块

### 3.1 布局架构 (Layout Architecture)

为支持双轨制，系统将引入多布局模式：

- **`PortalLayout.vue` (新增)**：C 端前台专用。包含：
  - **顶部导航栏 (Header)**：固定在顶部。左侧 Logo；中间为强大的全局搜索框（支持关键词、可能的颜色/比例筛选）；右侧为“上传”按钮、用户头像/下拉菜单（我的画板、退出登录）。
  - **主内容区 (Main Content)**：全屏宽度（或响应式最大宽度），主要承载瀑布流。
- **`MainLayout.vue` (保留/改造)**：B 端后台专用。保留现有的左侧菜单树和顶部工具栏结构。

### 3.2 核心页面规划 (Core Pages - MVP)

1.  **首页/发现页 (`/explore` 或 `/`)**
    - **职责**：展示全站审核通过的公共高质量素材。
    - **组件**：
      - 热门标签分类（如“UI设计”、“自然摄影”、“3D渲染”），横向滚动或排列在搜索框下方。
      - **核心组件**：支持无限滚动的**响应式瀑布流 (Masonry Grid)** 组件。
      - **排序/过滤**：按“最新发布”、“最受欢迎”切换。

2.  **搜索结果页 (`/search?q=xxx`)**
    - **职责**：展示基于关键词或高级筛选（如颜色主色调、横竖比例）的搜索结果。
    - **复用组件**：与首页复用瀑布流组件，顶部展示当前搜索词和筛选条件面板。

3.  **图片详情交互层 (Image Detail Modal / Page)**
    - **职责**：展示图片的详细信息并提供核心操作。
    - **展现形式**：
      - **浅层**：瀑布流中点击图片触发全屏/半屏模态框（Modal）。左侧展示大图，右侧展示信息。
      - **深层**：模态框内提供一个明显的“查看详情页”按钮，点击跳转至独立的 `/image/:id` 路由页面（便于分享和深度展示如：相似推荐、作者主页入口、完整的 EXIF 和色板）。
    - **信息项**：标题、作者、发布时间、尺寸/格式、标签。
    - **操作项**：**采集到画板 (核心)**、下载原图、点赞、分享链接。

4.  **个人中心 / 我的画板 (`/u/:username/boards`)**
    - **职责**：用户的私人灵感库和资产管理中心。
    - **页面结构**：
      - 顶部用户信息（头像、昵称、简介、统计数据）。
      - Tab 切换：“画板 (Boards)”、“我的上传 (Uploads)”、“赞过 (Likes)”。
      - **画板列表**：展示用户创建的画板卡片（通常显示该画板内的前 3-4 张缩略图封面）。
    - **画板详情 (`/board/:id`)**：点击某个画板进入，展示该画板内的图片瀑布流。
    - **上传入口**：用户在此处触发全局的图片上传组件，选择上传到哪个私有画板。

### 3.3 核心业务流：图片生命周期 (Image Lifecycle)

1.  **上传 (Upload)**：用户在“我的画板”或顶部导航点击上传 -> 拖拽图片 -> 填写标题/标签 -> 选择目标画板 -> 图片状态变为 `private`（私有）。
2.  **投稿 (Publish Request)**：用户在私有图片的操作菜单中点击“申请公开” -> 图片状态变为 `pending`（待审核）。
3.  **审核 (Review - B端功能)**：管理员在后台“图片审核”列表看到待审图片 -> 批准 (Approve) / 拒绝 (Reject)。
4.  **公开展示 (Public)**：审核通过后，图片状态变为 `public` -> 出现在 C 端首页的瀑布流和搜索结果中。
5.  **采集传播 (Pin)**：其他 C 端用户在瀑布流看到这张 `public` 图片 -> 点击“采集” -> 将图片的引用 (Reference/Link) 添加到自己的某个画板中（并非复制物理文件，而是建立关联）。

## 4. 技术选型建议 (Technical Stack Adjustments)

- **瀑布流组件**：由于 Element Plus 没有原生的瀑布流组件，建议引入专门的 Vue 3 瀑布流库（如 `vue-masonry`，或者自己基于 CSS Column / Grid 实现一个轻量级的响应式瀑布流）。
- **图片懒加载与骨架屏**：瀑布流强依赖图片的尺寸信息，建议在后端/Mock 数据中预先返回图片的宽高比（aspect ratio），前端利用占位符（Skeleton）或纯色块（提取的主色调）支撑布局，防止图片加载过程中的页面抖动 (Layout Shift)。
- **路由改造**：需要在 `src/router/` 中重新规划嵌套路由，区分 `/admin/...`（使用 `MainLayout`）和 `/...`（使用 `PortalLayout`）。

## 5. 后续迭代规划 (Future Scope)

- _这些功能不在 MVP 阶段，但属于素材图库的重要发展方向：_
- **图片色板提取**：前端或后端利用 `color-thief` 提取图片主色调，支持“按颜色搜图”。
- **相似图片推荐**：基于标签或图像特征的“猜你喜欢”算法。
- **在线裁切/轻编辑**：引入 `cropperjs`，允许用户在采集前或下载前对图片进行简单的裁剪。
- **多规格下载**：提供 WebP、高清、原图等多种格式和尺寸的下载选项。
